# Stage 0 第一次跑（2026-05-17）

## 配置

- 识别核：1 个（food_recognizer，识别 plant + cow）
- 规划核：1 个（greedy_nearest）
- Executor：确定性（非 LLM）—— 把 goto(node) 直接翻译成 move_*
- 视野：9×9（view_radius=4）
- 步数上限：200
- 3 个种子：0, 1, 2

## 关键结论

1. **`eat_cow`：Stage 0 = 3/3，Random = 0/3** —— 这是核心证据。"识别核 → 节点图 → 规划核 → goto" 这条 pipeline 确实让 agent 做出了**目标导向的觅食行为**，random 做不到。

2. **Stage 0 死得更快（seed=0 仅 48 步）**：单一目标的代价。agent 一心追牛，撞进 zombie 区域被打死。**这正好暴露了下一步必须做的事**：威胁识别核 + 避险规划核。

3. **Random 反而平均"活更久"是个假象**：random seed=1 活到 200 步是因为 200 步还没饿死（food=2 但还没到 0）；random 没死≠ agency，只是熵驱动让它没专心去送。

4. **附带成就（collect_wood / collect_sapling / collect_drink）是路上撞出来的**——agent 没识别这些目标，但移动时 `do` 触发了。这说明 Crafter 的成就解锁对偶发动作敏感，**对比时主指标应该是 eat_cow / eat_plant，附带成就是噪声**。

```
step 0:  视野: cow_E_mid               → goto cow_E_mid    → [move_right]
step 1:  视野: cow_E_mid               → goto cow_E_mid    → [move_right]
step 2:  视野: cow_NE_close            → goto cow_NE_close → [move_right]
```

节点 id 随 agent 移动稳定演变（mid → close → close），决策路径完全可读。
