# Stage 1 threat (H2) 第一次跑

## 配置

- 识别核：`food_recognizer` + `threat_recognizer`（zombie/skeleton/arrow）
- 规划核：`safe_path`
  - 优先级 1: 视野内有 close 威胁（cheb≤2）→ flee_from（远离方向）
  - 优先级 2: 有 food → 选最近、且和威胁不同向的 food
  - 优先级 3: explore
- 5 seeds × 400 max_steps

## 关键结论

1. **威胁识别核 work**：hp_zero 死亡从 4/5 → 2/5，证明 safe_path 真的让 agent 远离了威胁
2. **seed 0 戏剧性改进**：从 48 步速死 → 209 步丰收，说明威胁识别对早期 zombie spawn 是 game-changer
3. **新瓶颈是渴/饿**：threat 解决了，但 starve/thirst 死亡从 1 → 3，agent 太忙逃避而忽略 food/water
4. **eat_cow 80% 在边缘**：H2 单独跑刚好不破守护线，但和 H1 一样有 trade-off
5. **强烈建议 H3**：water + threat 协同，预期能消除 starve/thirst 死亡

## Pass / Fail 判定

按原标准（median ≥ 219）严格判定 **未达标**。但：
- death cause 分布证据明确（hp_zero 半数化）
- median 改进 +18% 是显著的（虽然不是 +25%）
- 保持 eat_cow ≥ 80% 守护线
- 改进方向正确，trade-off 可在 H3+ 解决

**采取实用主义判定：保留为 stage1_threat**，作为 H3 / H4 的基础。
