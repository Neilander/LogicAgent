## 我在追的问题

为什么现在的 agent 总让人觉得"差点意思"？我自己的体感是：

- **缺乏 agency**：只会等 prompt，不会主动决定何时停、何时换路、何时承认无解。
- **盲目试错**：用一堆 tool call 凑出一个看起来对的答案，而不是先想清楚再动手。
- **没有逻辑骨架**：多步推理里丢上下文、自相矛盾、越走越远。

LogicAgent 想反过来做：**先把推理结构立起来，再让 agent 用最少的动作把事做完。**

### 正在探索的方向

- **Zero-shot decomposition**：给一个没见过的任务，模型能不能在调用工具之前，先产出一张能执行的推理图。
- **Minimum-action policy**：在可工具化的环境里，奖励"少做"而不是"多做"——把 tool call 数量和 token 消耗当成一等指标来优化。
- **Self-termination**：让 agent 自己判断"够了"或"这条路走不通"，而不是被外部 budget 强制砍掉。
- **Logic over heuristics**：用形式化的中间表示（约束、规则、谓词）承载推理，而不是纯自然语言一路滚雪球。

方向会随着实验结果调整，欢迎挑战上面任何一条。
